# T14 — Notificaciones en app y push (opcional)

| Campo | Valor |

| **Historia(s)** | [SH1](../historias-usuario/SH1.md) |
| **Área** | Backend + Frontend |

## Descripción

Implementar el centro de notificaciones en el frontend (lista, marcar como leído, eliminar) consumiendo la API de notificaciones. Opcionalmente implementar el registro de dispositivo para push y la entrega de notificaciones push desde el backend (Firebase).

## Criterios de aceptación

- **AC1**: En el frontend hay un acceso al centro de notificaciones (icono, menú) que muestra la lista con `GET /api/notifications`; se pueden marcar como leídas y eliminar.
- **AC2**: El backend ya expone `GET /api/notifications`, `POST /api/notifications/{id}/read`, `POST /api/notifications/read-all`, `DELETE /api/notifications/{id}` (según rutas existentes).
- **AC3**: Jobs o comandos en backend que generen notificaciones para próximos mantenimientos, ITV y seguros (según reglas ya existentes).
- **AC4** (opcional): Registro de dispositivo push con `POST /api/push-notifications/register` y envío de notificaciones push cuando corresponda (Firebase); frontend con servicio worker y permisos.

## Notas técnicas

- Frontend: `NotificationCenter.jsx`, `usePushNotifications.js`. Backend: `NotificationController`, `PushNotificationController`, jobs de recordatorios.
