## Descripción

Implementar el dashboard (ruta `/`) que muestre el listado de vehículos de la empresa obtenido con `GET /api/vehicles`, con paginación y opcionalmente búsqueda. Cada elemento debe ser clicable y navegar al detalle del vehículo.

## Criterios de aceptación

- **AC1**: La ruta `/` (o `/dashboard`) muestra el listado de vehículos; los datos se obtienen con `GET /api/vehicles` (paginación con `page`, `per_page` si la API lo soporta).
- **AC2**: Cada vehículo muestra al menos: matrícula, marca/modelo (o nombre legible), km actual o último repostaje.
- **AC3**: Al hacer clic en un vehículo se navega a `/vehicle/{id}`.
- **AC4**: Hay un Layout común (cabecera, menú, pie) que se muestra en las rutas autenticadas; el dashboard está dentro de ese layout.
- **AC5**: Estados de carga (skeleton o spinner) y de error (mensaje si falla la petición); lista vacía mostrada con mensaje amigable.
- **AC6**: Opcional: búsqueda por matrícula/modelo usando query param `search` y filtro por marca (`brand_id`).

## Notas técnicas

- Página: src/pages/Dashboard.jsx en el frontend. Layout: Layout.jsx. API: api.get('/vehicles', { params: { page, per_page, search, brand_id } }).
