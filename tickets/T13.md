# T13 — Frontend: Historial e informe vehículo

## Descripción

Implementar la vista de historial del vehículo (repostajes y mantenimientos en listado ordenado por fecha) y la vista de informe (consumo L/100 km, coste por km y datos que devuelva `GET /api/vehicles/{id}/report`).

## Criterios de aceptación

- **AC1**: Ruta `/vehicle/:id/history` (o pestaña Historial dentro del detalle) que muestre repostajes y mantenimientos del vehículo; datos desde `GET /api/fuel-refills?vehicle_id=...` y `GET /api/maintenances` filtrado por vehicle_id (o endpoint equivalente).
- **AC2**: Listados ordenados por fecha (más reciente primero); cada ítem muestra información suficiente (fecha, km, litros/coste para repostajes; fecha, tipo, coste para mantenimientos).
- **AC3**: Ruta `/vehicle/:id/report` que llame a `GET /api/vehicles/{id}/report` y muestre: consumo medio (L/100 km), coste por km, y opcionalmente desglose o gráficos.
- **AC4**: Si no hay datos suficientes para el informe, mostrar mensaje "N/A" o "Insuficientes repostajes para calcular".
- **AC5**: Navegación clara desde el detalle del vehículo a Historial e Informe; breadcrumb o título con nombre del vehículo para contexto.

## Notas técnicas

- Páginas: `History.jsx`, `VehicleReport.jsx`. API: `api.get('/vehicles/' + id + '/report')`, `api.get('/fuel-refills', { params: { vehicle_id } })`, etc.
